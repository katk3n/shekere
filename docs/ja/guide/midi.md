# MIDI

Shekereは接続されたMIDIデバイスを自動的に検出し、ノート（音符）やコントロールチェンジ（CC）のデータをリアルタイムで提供します。

## `midi` オブジェクト

`midi` オブジェクトには、それぞれ128個の要素を持つ2つの配列が含まれています。すべての値は **0.0 〜 1.0** の範囲に正規化されています。

| プロパティ | 説明 |
| :--- | :--- |
| `midi.notes` | ノートのベロシティの配列。`midi.notes[60]` が中央のCです。 |
| `midi.cc` | コントロールチェンジ（ノブ、フェーダーなど）の配列。 |

### ノートデータ
- キーが押されると（Note On）、配列内の値はベロシティ（叩いた強さ）を127で割った値になります。
- キーを離すと（Note Off）、値は `0` に戻ります。

### CCデータ
- ノブやスライダーを動かすと、対応する `midi.cc` 配列のインデックスの値が `0.0` から `1.0` の間で更新されます。

## 使用例

### コントローラーをエフェクトにマッピングする
スライダー（多くのコントローラーで CC 74 に割り当てられています）を使ってブルームの強度を制御する例：

```javascript
export function update({ midi, bloom }) {
  // CC 74 (0.0 - 1.0) をブルーム強度に直接マッピング
  bloom.strength = midi.cc[74] * 3.0;
}
```

### キー入力でビジュアルをトリガーする
一番低いC（ノート番号36）が押されたときに色をフラッシュさせる例：

```javascript
export function update({ midi }) {
  const velocity = midi.notes[36];
  if (velocity > 0) {
    this.colorIntensity = velocity;
  }
  this.colorIntensity *= 0.95; // 徐々に消去
}
```

## MIDI使用のヒント

1.  **正規化**: Shekereの値は常に 0.0 〜 1.0 なので、MIDIデータを手動で127で割る必要はありません。
2.  **スムージング**: MIDIコントローラーは毎秒60フレームよりも低い頻度でデータを送信することが多いため、ビジュアルに「階段状」の動きが出ることがあります。CC値を滑らかにするには、次のようなイージングを検討してください：
    `currentValue += (midi.cc[10] - currentValue) * 0.1;`
3.  **デバッグ**: ハードウェアがどのCC番号やノート番号を送信しているか不明な場合は、Shekereの **コントロールパネル（Control Panel）** 内の **Monitors** セクションを確認してください。受信しているMIDI信号がリアルタイムで視覚的に表示されます。
