# エフェクト (ポストプロセッシング)

Shekereには強力なポストプロセッシング・パイプラインが組み込まれており、**コントロールパネルのUI** または **スケッチのコード** から直接制御できます。

## 双方向同期 (Bidirectional Sync)

Shekereのユニークな特徴の一つは、**「双方向同期」** です。
- コントロールパネルのスライダーを動かすと、コード内の対応する値が変化します。
- コード内で値を変更すると（例：`bloom.strength = 1.5`）、コントロールパネルのスライダーが自動的に追従して動きます。

これにより、手動での微調整とコードによる自動制御をシームレスに行き来することができます。

## 利用可能なエフェクト

`update` 関数に渡される `context` オブジェクトから以下のエフェクトにアクセスできます。

### 🌸 ブルーム (Bloom / 発光)
| プロパティ | 範囲 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| `bloom.strength` | 0.0 – 3.0 | `0.0` | 発光全体の強度。 |
| `bloom.radius` | 0.0 – 1.0 | `0.0` | ブルームのぼかし半径。 |
| `bloom.threshold` | 0.0 – 1.0 | `1.0` | 輝度の閾値。この値より明るいピクセルのみが発光します。 |

### 🌈 RGB シフト
| プロパティ | 範囲 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| `rgbShift.amount` | 0.0 – 0.05 | `0.0` | 色収差（カラーチャンネルのずれ）の量。 |

### 🎞️ フィルムグレイン (Film Grain)
| プロパティ | 範囲 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| `film.intensity` | 0.0 – 2.0 | `0.0` | ノイズ粒子の強度。 |

### 🎭 ヴィニエット (Vignette)
| プロパティ | 範囲 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| `vignette.offset` | 0.0 – 3.0 | `0.0` | ヴィニエット（四隅の暗がり）の半径。 |
| `vignette.darkness` | 0.0 – 3.0 | `1.0` | エッジの暗さの強度。 |

## 使用例

```javascript
export function update({ audio, bloom, rgbShift, film }) {
  // 低音（bass）に合わせてシーンを発光させる
  bloom.strength = audio.bass * 2.0;

  // 高音域に合わせてグリッチのようなRGBシフトを追加
  rgbShift.amount = audio.high * 0.01;

  // フィルムグレインの強度を固定
  film.intensity = 0.5;
}
```

## パフォーマンスについての注意

GPUの消費電力を抑えるため、Shekereは強度が `0` に設定されているポストプロセス・パスを自動的に無効化します。エフェクトを全く使用しないスケッチの場合、パフォーマンスへの影響はありません。
